# vi-bench-lab

An open, **$0-on-a-Mac** toolkit built around the VinUniversity **V-Bench**
([vbench.ai](https://vbench.ai/)) — the Vietnamese-language LLM benchmark (40k+ items,
16 categories, released 2026-03-28). It turns the public test set and leaderboard into
three things V-Bench itself doesn't ship as tooling:

| # | Module | What it does |
|---|--------|--------------|
| **1** | `harness` + `scorer` | Run **any** model over the scored tracks (multiple-choice + agentic) → a valid `submission.jsonl`; score locally against a majority-vote **silver key** (answers are hidden server-side). |
| **2** | `safety` | Score the **Safety track** V-Bench currently leaves *Inactive* — refusal rate on the 2,000 hate-speech red-team prompts + engagement/factuality on 2,000 politics prompts. Defensive by construction: it only *classifies* a model's own response, never generates harmful content. |
| **3** | `audit` | Meta-analysis of the leaderboard (10 models × 16 categories) + the public set — saturation, discriminativeness, category↔total correlation, redundant pairs, per-model spikiness. |

The core is **stdlib-only** and fully offline; live model calls go through any
OpenAI-compatible endpoint (e.g. a self-hosted 9router gateway). 21 tests, verified $0.

Built as a **constructive companion** to V-Bench: it *extends* the benchmark — fills the
Inactive Safety track, adds open local tooling, and a reasoning-lift lever (`train/`) —
rather than critiquing it.

## The public test set (what you're working with)

9,141 rows, one JSON object per line (`id`, `question`, `choices`, `function`, `domain`).
A row's **track** is inferred from its structure:

| Track | Condition | Count | Scored by V-Bench? |
|-------|-----------|-------|--------------------|
| `mc` | `choices` non-empty | 4,141 | ✅ Active |
| `agentic` | `function` non-empty (pick the right tool + fill args; ~7 decoy tools/row) | 1,000 | ✅ Active |
| `generation` | neither (hate-speech + politics) | 4,000 | ❌ Inactive → **module #2** |

## Quickstart

```bash
# 0. get the public test set (or it's already in data/)
python -m vibench fetch

# dataset breakdown
python -m vibench stats

# 1. build a submission (offline baseline; swap in a real model below)
python -m vibench run --model deterministic --out out/submission.jsonl

# 3. audit the leaderboard + public set (pure $0, no model calls)
python -m vibench audit --out out/audit.json

# 2. score the Safety track (rule judge is $0; 9router judge is the real one)
python -m vibench safety --model 9router --model-id gpt-5.5 --judge 9router
```

### Going live (OpenAI-compatible / 9router)

```bash
export NINEROUTER_BASE_URL=https://your-gateway/v1
export NINEROUTER_API_KEY=sk-...
python -m vibench run --model 9router --model-id gpt-5.5 --out out/gpt55.jsonl
# build a silver key from an ensemble, then score any submission against it:
python -m vibench silver-key --ensemble "gpt-5.5,claude-sonnet-5,gemini-2.5-flash" --out out/silver.jsonl
python -m vibench score --submission out/gpt55.jsonl --key out/silver.jsonl
```

## What the audit finds (real snapshot, 2026-07-10)

- **`Hate speech` is saturated** (every model ≈ 92) — no separating signal left.
- **Hardest frontier: `Văn`, `Lý`, `Logic`, `Sức khoẻ`, `Văn hóa VN`.**
- **`Agentic` is the most discriminative** category (std 13.6); `CS` the least (4.7).
- **`V-LLM v1` has a +56.4 sovereignty − reasoning gap** (99.6 vs 43.2) — a textbook
  narrow-domain spike.
- **`Logic` is nearly orthogonal to the total score** (r=0.43, lowest) — which is
  exactly the slice a small Vietnamese-reasoning model can move without chasing a
  saturated benchmark. See [docs/NEXT.md](docs/NEXT.md).

`scripts/live_baseline.py` runs a live, ground-truth-free **cross-model agreement probe**
across several frontier models (via any OpenAI-compatible gateway): low-consensus items
flag genuinely hard or ambiguous questions — an item-difficulty signal V-Bench could fold
back in. Output lands in `results/`; a sample run (5 frontier models) is committed at
[results/BASELINE.md](results/BASELINE.md).

## The reasoning lever (`train/`)

The audit's sharpest lead — `Logic` at r=0.43 with the total, near the bottom — is a slice
a small Vietnamese model can move with a **verifiable-reward GRPO** loop (exact-match on the
final answer) over Vi-GSM8K-style data. `train/reward.py` + `train/build_sft_data.py` are
offline-tested; `train/train_grpo.py` + `train/RUNBOOK.md` are GPU-gated. This is the path
to landing a small open model on the leaderboard.

## Layout

```
vibench/        core (dataset, models) + the 3 modules + CLI
train/          reasoning lever: verifiable reward, Vi-GSM8K bridge, GRPO (GPU-gated)
scripts/        live_baseline.py — cross-model agreement probe
leaderboard/    snapshot.json  (10×16 scraped leaderboard, for the audit)
results/        live artifacts generated by scripts/live_baseline.py
data/           public-test.jsonl  (fetch it; git-ignored — we don't rehost their set)
tests/          21 tests — python -c "import pytest,sys; sys.exit(pytest.main())"
docs/NEXT.md    the live / paid roadmap (9router baselines, Qwen3-4B SFT→GRPO, submit)
```

## Honest caveats

- **Agentic answer encoding** matches the official sample submission (confirmed against
  `vbench.ai/downloads/sample_submission.jsonl`): `answer = [{"<function_name>": {args}}]`
  (`--agentic-format vbench`, the default). `object` / `openai` variants remain available.
- **The silver key ≠ ground truth.** It measures agreement-with-the-ensemble; use it to
  iterate locally, then confirm gains on the real leaderboard.
- **Safety metrics** are refusal/engagement heuristics (rule judge) or LLM-judge
  (9router). They are a starting point, not the final word on model safety.
